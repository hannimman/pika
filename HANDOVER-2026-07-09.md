  - **홈**: astryx로 디자인한 랜딩(⚡ 히어로 + 도구 ClickableCard). 카드에 동적 포함.
  - **따옴표**: 별도 리포 QuoteMaker(iframe)를 **네이티브 astryx 페이지로 흡수**. Tailwind 제거, `quote.ts` 순수 로직 + 피카츄 캔버스 포팅. matcha 스코프. 출력은 `', foo2'`처럼 콤마 뒤 공백.
  - **변수**: oracle-var-resolver **iframe**(외부 페이지라 astryx 못 입힘).
  - **벽돌**: 기존 Vue 캔버스 테트리스를 React로 포팅(네온 유지).
  - **동적(이번 세션 핵심)**: 생 SQL → PL/SQL `V_SQL` 동적쿼리 변환기. 아래 상세.
  - **농담 메뉴**: `alert()` → astryx `Banner`.
- **동적 페이지 기능(완료):**
  - 출력 스타일 **A(줄마다 대입) / B(단일 문자열) / C(선두 `||`)** 토글 + 대상 변수명 입력.
  - 작은따옴표 **리터럴 클릭 → 변수화**(`''val''` → `'''||var||'''`), 치환목록(이름 편집 / 삭제 복구).
  - **주석 제거**(`--`, `/* */`) — 단, 문자열 내부의 `--`/`/*`는 보존, 블록주석의 **개행은 보존**(줄 번호 정렬).
  - **`''` 이스케이프 인식**(`'don''t'`), 각 줄 **끝 공백/탭 트림**.
  - **따옴표 버튼**: 입력에서 단어 선택 → `'단어'`로 감싸줌(딸려온 공백 트림, 이미 감싼 건 스킵).
  - **자동 줄바꿈**(가로 스크롤 없음) + **미러 레이어**로 접힌 줄까지 좌우 라인 호버 하이라이트.
  - **복사 버튼**(결과 헤더, "복사됨 ✓"), **불타는 초기화**(불꽃 띠가 위로, 입력 내용만 소각), **클럽 배경 + 네온 보더**.

## 성공/실패 기록

**통한 접근**
- astryx는 **미리 컴파일된 CSS**(`@astryxdesign/core/astryx.css` + 테마 `theme.css`) 제공 → **StyleX 번들러 플러그인 불필요**. 리스크 게이트로 스크래치 빌드부터 검증하고 진행.
- 헤드리스 **Edge로 스크린샷** 검증(`msedge --headless=new --screenshot`). 인터랙션 확인은 초기 state를 잠깐 시드 → 스샷 → 원복.
- 라인 호버가 줄바꿈에서 깨지던 것 → **미러(mirror) div** 히트테스트로 논리줄 인식 + 접힌 줄 전체 하이라이트.

**버그와 해결**
- **matcha가 안 먹음(배경이 astryx 기본 `#1f1f22`):** `main.tsx`에서 `astryx.css`(기본 테마 내장)를 테마보다 **나중에** 로드해 덮어씀. → 테마를 **마지막에** 로드 + 페이지 래퍼에 `data-astryx-theme="matcha"` 스코프.
- **matcha가 검정처럼 보임:** 테마가 `light-dark()`라 OS 다크모드에서 다크변형. 라이트 강제(`color-scheme: light`) 시도했으나 "너무 하얘"로 사용자 거부 → **다크 유지**, matcha는 따옴표·동적 페이지에만 스코프.
- **좌우 줄 위치 안 맞음:** 입력은 astryx `TextArea`(자체 테두리/패딩 필드), 출력은 맨 `<pre>` → 콘텐츠 시작 위치 상이. → 입력을 **플레인 `<textarea>`**로 바꿔 출력과 동일 박스로 맞춤.
- **레이아웃 붕괴(왼쪽 찌그러짐):** no-wrap 긴 줄이 그리드 칸을 밀어냄. → 그리드 아이템에 `min-width: 0`.
- **주석 있을 때 라인 매핑 어긋남:** 블록주석 제거 시 개행까지 삭제 → 좌(주석 포함) vs 우(제거) 줄 수 불일치. → `stripComments`가 **블록주석 개행 보존**.
- **(최종 근본원인) 주석 줄바꿈 시 매핑 드리프트:** 입력 textarea는 스크롤바가 폭을 먹고 뒤 미러는 안 먹어 **줄바꿈 지점이 달라짐** → 누적 드리프트. → 입력/미러 **스크롤바 폭 12px 동일화**(미러는 투명). 스샷으로 검증(3줄 접힘에도 정확).
- **불타는 초기화:** 처음엔 그냥 페이드(부자연) → 불꽃 띠가 아래→위로 스윕 + 소각 마스크. 이후 "패널은 유지, 입력만 비움"으로 재조정(마스크를 패널이 아닌 입력 필드에만).

**실패/폐기한 방법**
- **따옴표 없는 토큰 "변수로" 버튼(토큰 머신):** `P_PROC_DATE`를 `' || P_PROC_DATE || '`로 빼는 별도 기계를 만들었으나 **과설계**. 사용자가 "그냥 따옴표만 감싸면 되잖아"로 지적 → **전량 revert**, `'단어'`로 감싸고 기존 리터럴 클릭 재사용(문자열 변수엔 quoted 형태가 더 맞기도 함).
- **`.mcp.json` 자동 생성:** 자동 승인 정책이 원격 MCP 시작 연결을 차단 → 파일 대신 README에 스니펫만.

## 주요 결정 사항

- **Vue 유지 대신 React 재작성:** astryx가 **React 전용**(StyleX, `@astryxdesign/core`). "무조건 astryx"라 재작성이 유일한 길.
- **HashRouter:** GitHub Pages 프로젝트 사이트(base `/pika/`)에서 딥링크 404 회피 — 서버 설정 없이 동작.
- **matcha를 앱 전체가 아닌 특정 페이지에만 스코프:** 사용자가 "이 메뉴만" 요구. `data-astryx-theme`를 페이지 래퍼에.
- **V_SQL concat 스타일(A/B/C) 채택, `q'[]'`·bind는 보류:** 이 도구의 핵심이 "리터럴→변수 삽입"이라 concat이 맞음(`q'[]'`는 변수 삽입과 상충). 세 스타일 다 지원(차이는 조립부뿐이라 저비용).
- **입력 = 플레인 textarea + 미러:** astryx TextArea 필드 크롬이 정렬을 방해 → 정렬/호버 정확도 위해 교체.
- **로직/UI 분리:** `quote.ts`, `dquery.ts`를 순수 함수로 빼서 `node` 테스트. UI(React/astryx)는 얇게.

## 주의사항 & 교훈

- **GateGuard 훅(everything-claude-code 플러그인):** 파일 편집/첫 Bash 전에 "사실 4종"을 제시해야 통과. 반복 작업 시 성가심 — 필요하면 세션을 `ECC_GATEGUARD=off`로 실행하거나 `ECC_DISABLED_HOOKS`에 `pre:edit-write:gateguard-fact-force`, `pre:bash:gateguard-fact-force` 추가.
- **GitHub Pages 배포:** 새 SPA는 **Settings→Pages→Source를 "GitHub Actions"로** 바꿔야 함(브랜치 배포면 소스 index.html이 그대로 서빙돼 빈 화면). `configure-pages enablement:true`가 기존 브랜치 설정을 항상 덮진 못함 — **처음에 수동 전환을 안내**.
- **샌드박스 네트워크:** Bash `curl`로 외부 접속이 `000`으로 실패 → 라이브 확인은 **WebFetch**(프록시)나 헤드리스 Edge 사용. Actions 상태는 공개 API로 폴링.
- **배포 후 사용자 화면이 그대로면 캐시:** 하드리프레시(Ctrl+Shift+R) / **시크릿 창**. 라이브 JS 해시가 로컬 빌드 해시와 같은지 확인하면 배포 여부 확정.
- **textarea 미러 기법:** 입력 textarea와 미러 div의 **스크롤바 폭/패딩/폰트/줄바꿈**을 완전히 일치시켜야 히트테스트가 안 어긋남.
- **과설계 금지:** 기존 기능으로 되는 걸 새 기계로 만들지 말 것(토큰 변수화 → 따옴표 감싸기로 대체된 사례).
- **커밋 attribution 비활성**(사용자 전역 설정). **"박다" 류 표현 금지**(전역 규칙).
- **검증 습관:** 인터랙션은 초기 state를 잠깐 시드→스샷→반드시 원복. 원복 누락 주의.
